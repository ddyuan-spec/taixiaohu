泰小虎 - LLM意图识别Prompt V1.0

文档说明：本文档为LLM意图识别的完整Prompt设计，用于未来迭代时直接启用大模型意图识别功能。
> 当前状态：V1.0版本采用客户端意图选择方案，本文档作为备用保留。
> 启用条件：当需要更自然的对话体验、减少用户操作步骤时，可启用此LLM意图识别方案。

一、概述

1.1 功能定位

意图识别Prompt是整个对话链路的"交通指挥员"。它负责将用户的自然语言输入准确分类到不同的处理通道，确保后续Prompt能够精准响应。

1.2 在Coze工作流中的位置

用户输入 → [LLM意图识别节点] → 意图分类 → 路由到对应处理Prompt
                    ↑
              使用本文档Prompt

1.3 意图分类体系

意图类别标识符处理Prompt说明
健康咨询health_consultprompt_symptom_v1.0用户描述身体症状、询问健康问题
产品咨询product_consultprompt_product_v1.0用户询问保健品、营养品相关话题
健康知识knowledge_queryprompt_knowledge_v1.0用户询问健康科普知识、养生常识
其他otherprompt_boundary_v1.0闲聊、投诉、技术问题等非健康话题

二、完整Prompt内容

2.1 Prompt文本（prompt_intent_v1.0）

# 任务说明

你是一个意图分类器。你的任务是分析用户的输入内容，将其归类到以下四个类别之一。

# 分类定义

1. **health_consult**（健康咨询）：用户描述身体症状、询问健康问题、咨询身体状况相关话题
   - 关键词特征：不舒服、疼、痛、头晕、失眠、血压、血糖、咳嗽、拉肚子、胸闷、气短等
   - 示例特征："我最近总是头晕""我膝盖疼怎么办""血压高怎么办"

2. **product_consult**（产品咨询）：用户询问保健品、营养品、健康产品相关话题
   - 关键词特征：钙片、维生素、蛋白粉、鱼油、益生菌、买、多少钱、效果、哪个好等
   - 示例特征："你们有什么钙片推荐""鱼油怎么吃""这个产品多少钱"

3. **knowledge_query**（健康知识）：用户询问健康科普知识、养生常识、健康生活方式
   - 关键词特征：什么是、怎么预防、如何注意、饮食建议、运动建议、养生等
   - 示例特征："老年人怎么预防骨质疏松""吃什么对血压好""每天走多少步合适"

4. **other**（其他）：不属于以上三类的所有内容
   - 包括但不限于：闲聊、投诉、技术问题、非健康话题
   - 示例特征："今天天气怎么样""你们这个机器人怎么用""讲个笑话"

# 判断规则

1. 优先级：health_consult > product_consult > knowledge_query > other
2. 当用户同时涉及多个意图时，选择优先级最高的类别
3. 当用户描述症状并询问产品时，归类为health_consult
4. 当用户询问产品功效但未描述自身症状时，归类为product_consult
5. 当用户描述的症状可能涉及紧急情况时，在分类结果中标记urgent=true

# 输出格式

请严格按照以下JSON格式输出，不要输出任何其他内容：

{
  "intent": "分类结果（health_consult/product_consult/knowledge_query/other）",
  "confidence": "置信度（0.0-1.0）",
  "urgent": "是否紧急（true/false）",
  "keywords": ["提取的关键词1", "关键词2"],
  "reason": "分类理由（简短说明）"
}

2.2 输出格式定义

字段类型说明示例值
intentstring意图分类结果health_consult
confidencenumber置信度（0.0-1.0）0.95
urgentboolean是否紧急false
keywordsarray[string]提取的关键词["头晕", "血压"]
reasonstring分类理由用户描述了头晕症状

三、示例集

3.1 健康咨询示例

输入："我这两天头总是晕晕的，站起来的时候更明显"

输出：

{
  "intent": "health_consult",
  "confidence": 0.98,
  "urgent": false,
  "keywords": ["头晕", "站起来"],
  "reason": "用户描述了具体的身体症状（头晕），属于健康咨询"
}

3.2 产品咨询示例

输入："我想给我老伴买点钙片，你们这里有什么好的推荐吗"

输出：

{
  "intent": "product_consult",
  "confidence": 0.96,
  "urgent": false,
  "keywords": ["钙片", "买", "推荐"],
  "reason": "用户明确表达了购买钙片的需求，属于产品咨询"
}

3.3 健康知识示例

输入："老年人每天走路锻炼好不好啊？走多少步合适？"

输出：

{
  "intent": "knowledge_query",
  "confidence": 0.94,
  "urgent": false,
  "keywords": ["老年人", "走路", "锻炼", "步数"],
  "reason": "用户询问的是健康生活方式建议，属于健康知识类"
}

3.4 紧急健康咨询示例

输入："我老伴刚才突然说胸口特别疼，出了一身冷汗"

输出：

{
  "intent": "health_consult",
  "confidence": 0.99,
  "urgent": true,
  "keywords": ["胸口疼", "冷汗", "突然"],
  "reason": "用户描述了胸痛伴随冷汗的紧急症状，标记为紧急"
}

3.5 其他意图示例

输入："这个手机上的字太小了，我怎么把它调大一点啊"

输出：

{
  "intent": "other",
  "confidence": 0.92,
  "urgent": false,
  "keywords": ["手机", "字太小", "调大"],
  "reason": "用户询问的是手机操作问题，不属于健康相关话题"
}

4.3 紧急处理分支

当 urgent=true 时，无论intent为何，都需要：

  • 立即输出紧急提示话术
  • 建议用户尽快就医
  • 记录到紧急Case队列
