🐯 泰小虎 智能健康导购助手

产品需求文档（PRD）

版本：V1.3.0 状态：已确认 编制日期：2026-05-25
## 一、背景 ### 1.1 业务背景 当前中老年群体健康意识逐渐增强，但健康知识储备不足，对复杂数字化产品适应能力差。保健品市场规模持续增长，但消费者缺乏专业指导，存在误购、过度营养等问题。传统的电商平台和药店导购模式无法满足中老年用户的个性化健康需求。 本产品旨在构建一个基于大模型的智能健康导购助手——**泰小虎**，通过对话式交互，为中老年用户提供健康评估、症状自查、保健品推荐和健康知识科普服务。 ### 1.2 为什么用大模型解决
对比维度传统方案大模型方案
症状理解关键词匹配，无法理解自然语言描述理解自然语言描述，支持模糊表述
个性化推荐基于规则的统一推荐结合用户画像+健康风险动态推荐
知识库解答关键词检索，准确率低RAG + 大模型语义理解，准确率高
交互体验菜单式操作，步骤多对话式交互，贴近自然沟通
维护成本规则维护成本高Prompt调整即可，维护成本低
核心优势：大模型能够理解中老年用户的模糊、非标准化表述，这是传统规则引擎无法做到的。同时RAG技术能够确保健康知识和产品信息的准确性，避免幻觉问题。
### 1.3 竞品分析
竞品优势劣势泰小虎差异化
丁香医生医疗资质背书，品牌信任度高非导购型产品，无保健品推荐专注中老年+导购场景
平安好医生B端资源丰富产品复杂，中老年使用门槛高UI极简，语音交互优先
小红书健康内容生态好无AI导购能力，信息不精准RAG确保知识准确性
### 1.4 产品目标 **业务目标** - V1.0（MVP）：上线核心功能，验证产品-PMF契合度 - 目标用户数：V1.0上线3个月内达到 1000-5000人 - 转化率目标：导购转化率 3%-5% **模型目标** - 健康知识问答准确率 ≥ 90% - 产品推荐相关度 ≥ 80% - 用户对话满意度 ≥ 4.0/5.0 - 意图选择转化率 ≥ 85% ## 二、需求描述 ### 2.1 需求清单

### 2.2 需求分类 **功能需求** - 自然语言理解：支持中文自然语言输入，能够理解中老年用户的方言、模糊描述 - 多轮对话：支持追问式交互，如症状自查的逐步引导 - 知识检索：基于RAG的健康知识和产品信息检索 - 智能推荐：基于用户健康画像的保健品推荐 - 分群策略：按健康状态/行为意图对用户分群，差异化服务 **客户分群维度：** - 按年龄段：45-55岁、55-65岁、65岁以上 - 按行为意图：主动咨询型、浏览型、购买转化型 - 按健康状态：心血管风险、骨骼关节、睡眠问题、免疫力等 - 按消费能力：高消费能力、价格敏感、品牌忠诚 **后台管理：** 知识库管理、Agent配置管理、用户管理、数据看板、对话监控、Case下载 **性能需求** - 首次响应时间：简单问答 ≤ 2s，复杂推理 ≤ 5s - 并发支持：支持 50-200 并发用户数 - 可用性：99.9% **安全需求** - 用户健康数据加密存储，符合《个人信息保护法》 - 健康数据不出境，模型推理在国内完成 - 免责声明：每次症状自查结果必须包含免责提示 **数据需求** - 埋点数据：用户操作链路、对话内容、推荐点击/购买转化 - 正/case：用户点赞/购买=正case；用户反馈不准确/投诉=负case - 业务数据：日活跃用户、转化率、客单价、用户留存率 ## 三、业务流程图 用户视角的核心操作流程：
  1. 用户进入移动APP
  2. AI主动打招呼（语音+文字）
  3. 用户在APP界面选择意图类型
  4. 系统根据用户选择路由到对应工作流（健康咨询/产品咨询/其他）
  5. 健康咨询路径：症状追问 → 风险评估 → 推荐建议 → 引导就医
  6. 产品咨询路径：需求理解 → 产品推荐 → 购买引导

### 3.1 购买记录关联流程
流程环节说明
用户进入查询用户进入APP时，系统自动查询该用户的历史购买记录，加载至会话上下文
健康咨询关联健康咨询时，AI自动关联用户历史购买产品，提供个性化建议
复购提示根据购买周期，在适当时机提示用户复购（如保健品即将用完时）
订单问题转客服涉及订单查询、退换货、物流等问题时，自动转接人工客服
### 3.2 用户画像补全流程
流程环节说明
画像完整度检测系统实时检测用户画像完整度，识别缺失的关键信息字段
追问触发条件当对话涉及健康评估或产品推荐时，若画像不完整则触发追问
追问优先级过敏史 > 慢性病 > 年龄/性别，按优先级依次追问
关联人画像追问识别"给老伴买""给父母买"等场景，创建关联人画像并追问相关信息
### 3.3 复购推荐流程
流程环节说明
复购场景识别根据购买周期、用户主动询问、产品用完提醒等场景识别复购需求
历史产品推荐优先推荐用户历史购买过的产品，同时推荐相关联的补充产品
复购优惠说明展示复购专属优惠（如会员折扣、满减活动、赠品等），提升复购转化率
### 3.4 画像变更管理流程
流程环节说明
重大变更检测检测过敏史、慢性病、用药情况等关键字段的变更
变更记录存储所有画像变更记录存档，包含变更时间、变更内容、变更来源
风险提示重大变更时，系统自动评估对推荐结果的影响，必要时重新推荐并提示用户

## 五、模型选型 ### 5.1 选型约束条件
约束维度要求
数据安全用户健康数据不出境
成本控制MVP阶段月成本 5000-20000元
响应速度简单问答小于等于2s
部署方式混合部署（核心数据私有化，LLM推理走云端API）
### 5.2 可选模型对比
模型参数量中文能力部署方式价格适合场景
通义千问GLM-49B优秀API低意图识别+对话
Qwen2.5-72B72B优秀API中复杂推理
Qwen2.5-7B7B良好私有化自运营用户意图选择
文心一言4.0未公开优秀API低MVP快速验证
DeepSeek-V3685B MoE优秀API极低成本敏感
### 5.3 选型结论
选型方案：
  • 基于扣子（coze.cn）平台，使用Coze工作流和对话流
  • 主模型：使用Coze平台推荐的中文大模型
  • 备用模型：通过Coze平台failover机制切换
  • 成本预估：基于目标用户数1000-5000人，结合Coze平台定价估算

## 七、训练数据集（RAG知识库） ### 7.1 知识库内容
知识库内容说明数据来源更新频率

## 十一、AI能力边界说明 **能做到：** - 理解中文自然语言描述的健康问题 - 基于RAG提供准确的健康知识和产品信息 - 根据健康评估推荐合适的保健品 - 多轮对话引导用户描述症状 **做不到：** - 不能给出医疗诊断或用药建议 - 不能处理急诊情况（如胸痛，大出血等） - 无法保证100%准确率，可能存在偏差 - 不支持图片识别（V1.0） **已知缺陷：** - V1.0 暂不支持对接可穿戴设备数据 - V1.0 暂不支持体检报告解析 ## 十二、数据飞轮与迭代机制 ### 12.1 迭代闭环流程
