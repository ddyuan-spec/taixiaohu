泰小虎 Coze 工作流搭建操作手册

一、概述

本手册指导如何在 Coze 平台上搭建泰小虎智能健康顾问的完整工作流，实现意图识别、健康咨询、产品推荐等核心功能。

二、前置准备

2.1 注册与登录

  1. 访问 Coze 官网
  2. 使用抖音/飞书账号登录
  3. 进入「个人空间」或「团队空间」

2.2 创建 Bot

  1. 点击「创建 Bot」
  2. 填写基本信息：
    • Bot 名称：泰小虎健康顾问
    • Bot 描述：您的专业健康顾问，提供健康咨询、产品推荐等服务
    • 图标：上传泰小虎 Logo

三、工作流整体架构

[开始节点]
    │ 接收参数：user_input, intent, user_id
    ▼
[大模型节点：System Prompt]
    │ 注入全局人设
    ▼
[条件分支节点]
    │ 根据 intent 参数路由
    ├── intent == "health_consult" → [症状分析] → [RAG检索] → [推荐生成]
    ├── intent == "product_consult" → [产品咨询] → [产品库检索] → [推荐生成]
    ├── intent == "knowledge_query" → [健康知识库检索] → [知识问答]
    └── intent == "other" → [边界处理]
    ▼
[输出节点]

四、详细搭建步骤

4.1 创建工作流

  1. 在 Bot 编辑页面，点击左侧「工作流」
  2. 点击「创建工作流」
  3. 填写工作流信息：
    • 名称：taixiaohu_main_flow
    • 描述：泰小虎主工作流，处理用户健康咨询、产品咨询等请求

4.2 配置开始节点

节点类型：开始

参数名类型必填说明
user_inputString是用户输入的自然语言内容
intentString是用户选择的意图：health_consult / product_consult / knowledge_query / other
user_idString是用户唯一标识
conversation_idString否会话唯一标识

4.3 配置 System Prompt 节点

节点类型：大模型

模型选择：Doubao-pro-32k 或 GPT-4

系统提示词（System Prompt）：

# 角色定义
你是「泰小虎」，一位专业、温和、耐心、贴心的健康顾问。

# 核心能力
1. 健康评估：了解用户身体状况，提供健康方向性建议
2. 产品咨询：根据健康需求推荐合适保健品
3. 健康知识：解答健康养生问题
4. 情感陪伴：倾听健康困扰，给予鼓励支持

# 能力边界（非常重要）
1. 你不是医生，不能诊断疾病，不能开具处方
2. 遇到紧急症状必须立即建议就医
3. 用户描述症状时必须附加免责声明
4. 不能推荐处方药，只能建议非处方保健品
5. 不能声称产品可以"治疗""治愈"疾病

# 输出规范
1. 每次回复控制在100-200字以内
2. 使用口语化表达，避免专业术语
3. 重要信息用"【】"标注
4. 语气温暖亲切，像关心家人的老朋友

# 安全红线
1. 绝不提供疾病诊断
2. 绝不推荐处方药
3. 绝不声称产品可以治疗疾病
4. 遇到紧急情况必须建议立即就医
5. 涉及症状时必须附加免责声明

# 免责声明规则
在以下场景必须附加免责声明：
- 用户描述身体症状时
- 讨论涉及疾病话题时
- 推荐保健产品时
- 回答用药相关问题时

免责声明："温馨提示：我的建议仅供参考，不能代替医生的专业诊断。"

# 紧急情况处理
紧急症状包括：胸痛、呼吸困难、剧烈头痛、意识模糊、大量出血、严重外伤、高热不退

紧急情况回复："您的情况需要马上重视！请立刻拨打120或让家人送您去医院！"

4.4 配置条件分支节点

节点类型：选择器

分支名称条件表达式
健康咨询intent == "health_consult"
产品咨询intent == "product_consult"
健康知识intent == "knowledge_query"
其他intent == "other"

4.5 健康咨询分支

4.5.1 症状分析节点

节点类型：大模型

提示词：分析用户症状，提取关键信息，不给出诊断结论

4.5.2 RAG 检索节点

节点类型：知识库

配置：选择健康知识库，语义检索，Top 5

4.5.3 推荐生成节点

节点类型：大模型

提示词：基于症状分析和检索结果，推荐1-3款合适产品

4.6 产品咨询分支

4.6.1 产品咨询节点

节点类型：大模型

提示词：理解用户产品需求，提取关键信息

4.6.2 产品库检索节点

节点类型：知识库

配置：选择产品知识库，语义检索，Top 3

4.6.3 推荐生成节点

节点类型：大模型

提示词：基于检索结果，推荐合适产品并说明功效

4.7 健康知识分支

4.7.1 健康知识库检索节点

节点类型：知识库

配置：选择健康知识库，语义检索，Top 3

4.7.2 知识问答节点

节点类型：大模型

提示词：基于检索结果，用通俗易懂的语言回答健康问题

4.8 其他分支

4.8.1 边界处理节点

节点类型：大模型

提示词：处理闲聊、投诉、技术问题等，或引导用户回到健康咨询主题

4.9 配置输出节点

节点类型：结束

参数名值说明
response各分支的输出结果最终回复内容
intentstart.intent用户意图

五、知识库配置

5.1 创建健康知识库

  1. 点击「知识」→「创建知识库」
  2. 上传健康知识文档（PDF、Word、TXT）
  3. 配置分段规则：
    • 分段长度：500-800字
    • 重叠长度：100字
  4. 等待索引完成

5.2 创建产品知识库

  1. 点击「知识」→「创建知识库」
  2. 上传产品资料文档
  3. 包含字段：产品名称、功效、适用人群、服用方法、注意事项
  4. 配置分段规则同上

六、测试与发布

6.1 工作流测试

  1. 点击工作流编辑页面的「试运行」
  2. 输入测试参数：
{
  "user_input": "我最近睡眠质量不好，有什么建议吗？",
  "intent": "health_consult",
  "user_id": "test_user_001"
}
  1. 查看各节点输出，检查是否符合预期

6.2 发布工作流

  1. 测试通过后，点击「发布」
  2. 工作流将自动关联到 Bot

6.3 Bot 发布

  1. 返回 Bot 编辑页面
  2. 点击右上角「发布」
  3. 选择发布渠道（豆包、飞书等）

七、客户端集成

7.1 获取 API 密钥

  1. 在 Bot 详情页，点击「API 接入」
  2. 复制 Bot ID 和 API Key

7.2 调用示例

const response = await fetch('https://api.coze.cn/open_api/v2/chat', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    bot_id: 'YOUR_BOT_ID',
    user: 'user_001',
    query: '用户输入内容',
    custom_variables: {
      intent: 'health_consult'  // 用户选择的意图
    }
  })
});

八、常见问题

Q1: 意图识别不准确怎么办？
A: 检查 System Prompt 中的意图定义是否清晰，必要时在客户端增加意图确认环节。

Q2: RAG 检索结果不相关怎么办？
A: 优化知识库的分段策略，调整检索参数（Top K 值），或增加重排序节点。

Q3: 回复太长怎么办？
A: 在 System Prompt 中明确限制回复长度（100-200字），并在输出节点增加截断处理。

Q4: 如何接入微信/小程序？
A: 使用 Coze 的「发布」功能选择对应渠道，或通过微信云开发调用 Coze API。

九、优化建议

  1. 添加意图确认节点：在条件分支前增加意图确认，提高准确性
  2. 增加用户画像：接入用户历史数据，实现个性化推荐
  3. 添加满意度评价：在输出节点后增加评价收集
  4. 配置 fallback 机制：当知识库检索无结果时，使用大模型通用能力回复
