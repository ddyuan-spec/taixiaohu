## 1. 项目概述

### 1.1 产品定位

泰小虎是一款面向45-70岁中老年用户的AI健康导购助手，以移动APP为载体，通过对话式交互为用户提供健康评估、症状自查、保健品推荐和健康知识科普服务。产品基于扣子（coze.cn）平台构建AI能力，结合移动端生态实现从健康咨询到产品购买的全链路闭环。

### 1.2 核心价值

- **健康自查**：通过对话式症状询问，帮助用户初步了解自身健康状况
- **科学推荐**：基于用户健康画像，提供个性化的保健品推荐
- **便捷购买**：APP内完成从咨询到购买的全流程
- **语音优先**：支持语音输入与语音播报，降低中老年用户使用门槛

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| 年龄段 | 45-70岁 |
| 技术水平 | 智能手机基础使用者，对复杂操作不熟悉 |
| 健康需求 | 关注慢性病管理、日常保健、养生调理 |
| 消费特征 | 注重性价比，信任熟人推荐，对健康投入意愿强 |
| 使用场景 | 居家、社区、公园等，碎片化时间使用 |

### 1.4 功能架构

```
泰小虎APP
├── 启动/欢迎模块
│   └── 品牌展示 + 授权引导
├── AI对话模块（核心）
│   ├── 文字输入
│   ├── 语音输入
│   ├── TTS语音播报
│   ├── 等待动画
│   └── 用户反馈（点赞/踩）
├── 健康管理模块
│   ├── 健康问卷
│   ├── 健康报告
│   └── 健康档案
├── 产品导购模块
│   ├── 产品列表
│   ├── 产品详情
│   └── 订单确认/支付
└── 个人中心模块
    ├── 用户信息
    ├── 健康档案
    ├── 订单记录
    └── 设置
```

## 2. 设计原则与规范

### 2.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **极简优先** | 每个页面只聚焦一个核心任务，避免信息过载 |
| **大字体** | 最小字号16px，正文18-20px，标题24-28px |
| **高对比度** | 文字与背景对比度不低于4.5:1（WCAG AA标准） |
| **语音优先** | 关键交互支持语音输入，AI回复支持语音播报 |
| **容错设计** | 操作可撤销，重要操作有二次确认 |
| **渐进引导** | 首次使用提供引导提示，降低学习成本 |

### 2.2 色彩规范

#### 主色调

| 用途 | 色值 | 说明 |
|------|------|------|
| 品牌主色 | `#FF6B35`（活力橙） | 传达温暖、健康、活力，用于主按钮、品牌标识 |
| 品牌辅助色 | `#2D9D78`（健康绿） | 传达健康、安全、信任，用于健康相关模块 |
| 功能色-警告 | `#E8A317`（提醒黄） | 用于注意事项、不确定提示 |
| 功能色-危险 | `#D94F4F`（警示红） | 用于错误提示、重要警告 |
| 功能色-成功 | `#4CAF50`（成功绿） | 用于成功状态、正面反馈 |

#### 文字色

| 用途 | 色值 | 说明 |
|------|------|------|
| 主文字 | `#1A1A1A` | 标题、正文，高对比度 |
| 次文字 | `#4A4A4A` | 辅助说明、次要信息 |
| 占位文字 | `#999999` | 输入框占位符 |
| 反白文字 | `#FFFFFF` | 深色背景上的文字 |

#### 背景色

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#F5F5F5` | 浅灰，减少视觉疲劳 |
| 卡片背景 | `#FFFFFF` | 白色，内容区域 |
| AI气泡背景 | `#FFFFFF` | 白色，AI消息气泡 |
| 用户气泡背景 | `#FF6B35` | 品牌橙，用户消息气泡 |

### 2.3 字体规范

| 元素 | 字号 | 字重 | 行高 | 说明 |
|------|------|------|------|------|
| 大标题 | 28px | Bold (700) | 1.3 | 页面主标题 |
| 小标题 | 24px | Bold (700) | 1.3 | 模块标题 |
| 正文 | 20px | Regular (400) | 1.6 | 主要内容文字 |
| 辅助文字 | 18px | Regular (400) | 1.5 | 次要说明文字 |
| 最小文字 | 16px | Regular (400) | 1.4 | 标签、备注（尽量少用） |
| 按钮文字 | 20px | Medium (500) | 1.0 | 按钮内文字 |

### 2.4 间距规范

| 用途 | 数值 | 说明 |
|------|------|------|
| 页面左右边距 | 24px | 内容区与屏幕边缘的距离 |
| 卡片内边距 | 24px | 卡片内容与卡片边缘的距离 |
| 模块间距 | 20px | 不同模块之间的间距 |
| 元素间距 | 12px | 同一模块内元素之间的间距 |
| 按钮高度 | 56px | 主要操作按钮 |
| 按钮圆角 | 12px | 按钮圆角半径 |
| 卡片圆角 | 16px | 卡片圆角半径 |

### 2.5 组件规范

#### 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| 主按钮 | 橙色填充（`#FF6B35`），白字，高度56px | 主要操作（开始对话、立即购买等） |
| 次按钮 | 白色填充，橙色边框，橙字，高度56px | 次要操作（查看更多、稍后再说等） |
| 文字按钮 | 无背景，橙色文字，高度44px | 辅助操作（跳过、了解更多等） |
| 禁用按钮 | 灰色填充（`#CCCCCC`），灰字 | 不可用状态 |

#### 输入框

- 高度：56px
- 圆角：12px
- 边框：1px solid `#DDDDDD`
- 聚焦边框：2px solid `#FF6B35`
- 字号：20px
- 占位字号：18px

#### 卡片

- 背景色：`#FFFFFF`
- 圆角：16px
- 阴影：0 2px 12px rgba(0,0,0,0.08)
- 内边距：24px

## 3. 全局导航结构

### 3.1 底部Tab导航（4个Tab）

| Tab | 图标 | 名称 | 说明 |
|-----|------|------|------|

### 3.2 页面跳转关系

```
启动页 ──→ 意图选择页 ──→ 首页（AI对话页）
                              │
                              ├──→ 健康问卷页 ──→ 健康报告页 ──→ 产品详情页
                              │                                    │
                              ├──→ 产品列表页 ──→ 产品详情页 ──→ 订单确认页
                              │                                    │
                              ├──→ 个人中心页 ──→ 设置页
                              │                   ├──→ 健康档案
                              │                   └──→ 订单记录
                              │
                              └──→ AI对话页
                                    │
                                    ├──→ 意图确认弹窗（底部滑出）
                                    │           │
                                    │           ├──→ 意图选择弹窗
                                    │           │       │
                                    │           │       └──→ 更新意图，AI重新开场

                                    └──→ 意图切换弹窗（顶部标签进入）
```

### 3.3 全局组件

- **顶部状态栏**：APP原生状态栏，适配iOS/Android不同机型
- **导航栏**：页面标题 + 返回按钮（二级页面），高度44px
- **底部Tab栏**：4个Tab，高度56px（含安全区域适配）
- **全局Toast提示**：居中弹出，2秒自动消失，字号18px
- **全局Loading**：居中显示，品牌橙色加载动画
- **语音播报指示器**：页面顶部横幅，显示当前播报内容，可点击暂停

#### 交互说明

| 交互动作 | 触发条件 | 行为描述 |

| 已有账号登录 | 点击"已有账号？立即登录" | 跳转至账号密码登录页（子页面） |
| 页面自动跳转 | 启动页停留3秒后 | 若已登录，自动跳转至首页；若未登录，停留在欢迎页 |

| 元素 | 规格 | 说明 |

#### 注意事项

- 启动页背景色使用浅暖色调（`#FFF8F3`），营造温暖氛围
- 品牌形象要突出"健康""专业""温暖"三个关键词

## 5. 用户分群策略

### 5.1 分群维度

泰小虎基于以下四个维度对用户进行分群，实现精准推荐和个性化服务。

#### 维度一：年龄段

| 分群 | 年龄范围 | 特征 |
|------|----------|------|
| 中年早期 | 45-50岁 | 事业稳定期，开始关注健康，预防意识强 |
| 中年中期 | 51-55岁 | 慢性病初现，健康管理需求增加 |
| 中年后期 | 56-60岁 | 退休前后，健康问题增多，时间充裕 |
| 初老期 | 61-65岁 | 慢性病管理为主，养生意识强 |
| 老年期 | 66-70岁 | 多重健康问题，需要综合健康管理 |

#### 维度二：行为意图

| 分群 | 行为特征 | 推荐策略 |
|------|----------|----------|
| 健康咨询型 | 频繁使用AI对话，主动询问健康问题 | 强化AI对话体验，推送健康知识 |
| 症状自查型 | 主要使用健康自查功能 | 引导完善健康档案，推荐相关产品 |
| 购物导向型 | 直接浏览产品，购买频率高 | 优化购物体验，推送促销信息 |
| 内容浏览型 | 浏览健康知识，互动少 | 推送优质内容，引导使用AI功能 |
| 休眠用户 | 注册后很少使用 | 定期推送唤醒消息，健康提醒 |

#### 维度三：健康状态

| 分群 | 健康特征 | 推荐策略 |
|------|----------|----------|
| 健康良好 | 无慢性病，评分90+ | 预防性保健品，健康知识科普 |
| 轻度关注 | 1-2项指标偏高，评分75-89 | 针对性保健品，定期复查提醒 |
| 需要关注 | 多项指标异常，评分60-74 | 专业健康建议，相关产品推荐 |
| 重点管理 | 慢性病确诊，评分<60 | 用药提醒，医生建议，谨慎推荐 |

#### 维度四：消费能力

| 分群 | 消费特征 | 推荐策略 |
|------|----------|----------|
| 价格敏感 | 偏好低价产品，关注促销 | 推送优惠活动，性价比产品 |
| 品质导向 | 注重品牌和品质 | 推荐品牌产品，品质保障 |
| 高端消费 | 不敏感价格，追求最好 | 推荐高端产品，VIP服务 |
| 理性消费 | 货比三家，注重评价 | 提供详细对比，真实评价 |

### 5.2 分群应用场景

| 场景 | 应用方式 |
|------|----------|
| 首页推荐 | 根据用户分群展示不同的推荐内容和功能入口 |
| 产品排序 | 根据消费能力和健康状态调整产品推荐顺序 |
| AI对话 | 根据行为意图和健康状态调整对话策略 |
| 消息推送 | 根据用户分群发送个性化推送消息 |
| 健康报告 | 根据年龄段和健康状态调整报告内容和措辞 |

## 6. AI交互特殊规范

### 6.1 等待动画规范

| 场景 | 动画类型 | 显示内容 |
|------|----------|----------|
| AI思考中（<5秒） | 三个圆点跳动 | "泰小虎正在思考..." |
| AI思考中（5-10秒） | 圆点跳动 + 文字变化 | "正在认真分析您的问题..." |
| AI思考中（10-30秒） | 加载动画 | "泰小虎正在认真分析，请稍候..." |
| AI思考中（>30秒） | 加载动画 + 推荐内容 | "响应时间较长，您可以先看看以下内容" |

### 6.2 不确定性提示规范

以下场景必须在AI回复中添加不确定性提示：

| 场景 | 提示内容 |
|------|----------|
| 症状分析 | "以上分析仅供参考，建议您咨询专业医生获取准确诊断。" |
| 产品推荐 | "产品推荐基于您的健康信息，建议在医生指导下使用。" |
| 健康建议 | "以上建议仅供参考，请根据自身情况酌情采纳。" |
| 用药相关 | "AI不能提供用药建议，请咨询专业医生或药师。" |

### 6.3 用户反馈机制

| 反馈类型 | 触发方式 | 用途 |
|----------|----------|------|
| 点赞 | 点击点赞按钮 | 标记有用回复，优化AI模型 |
| 点踩 | 点击点踩按钮 | 标记无用回复，收集改进方向 |
| 补充反馈 | 点踩后弹出输入框 | 收集具体问题，人工审核改进 |

### 6.4 语音交互规范

| 规范项 | 说明 |
|--------|------|
| 默认状态 | 语音播报默认开启，语音输入默认开启 |
| 播报范围 | AI回复消息默认播报，用户消息不播报 |
| 播报控制 | 每条消息独立播放按钮，全局开关在导航栏 |
| 长文本处理 | 超过100字的回复，播报前3句话 + "详细内容请查看文字" |
| 播报速度 | 默认慢速（适合中老年用户），可在设置中调节 |
| 识别反馈 | 语音识别结果展示给用户，用户可编辑后发送 |
| 录音时长 | 单次录音最长60秒，超时自动结束 |
| 错误处理 | 识别失败时提示"没有听清，请再说一次" |

### 6.5 意图引导话术规范

用户选择不同意图进入AI对话时，AI使用对应的引导话术开场，帮助用户明确当前对话方向。

#### 6.5.1 健康咨询意图

**使用场景：** 用户选择"健康咨询"意图进入对话

**引导话术：**
> "您好！请问今天想咨询什么健康问题呢？比如哪里不舒服、想了解什么症状？"

**话术说明：**
- 语气亲切，使用"您"称呼
- 直接询问健康问题，降低用户开口门槛
- 提供具体示例（"哪里不舒服"、"想了解什么症状"），帮助用户组织语言
- 适合中老年用户的口语化表达

#### 6.5.2 产品咨询意图

**使用场景：** 用户选择"产品咨询"意图进入对话

**引导话术：**
> "欢迎了解健康产品！请问您想为哪方面的健康需求选购产品呢？"

**话术说明：**
- 使用"欢迎"营造热情氛围
- 强调"健康需求"而非直接推销产品
- 询问具体需求方向，便于后续精准推荐
- 可配合追问："比如是调节血脂、补钙、还是改善睡眠呢？"

#### 6.5.3 健康知识意图

**使用场景：** 用户选择"健康知识"意图进入对话

**引导话术：**
> "想学习健康知识吗？请问您对什么健康话题感兴趣？"

**话术说明：**
- 使用"学习"体现知识获取的积极意义
- 开放式提问，让用户主动表达兴趣方向
- 可配合推荐热门话题："比如养生食疗、运动保健、慢性病预防等"
- 语气轻松，适合科普场景

#### 6.5.4 其他问题意图

**使用场景：** 用户选择"其他问题"意图进入对话

**引导话术：**
> "请问有什么可以帮您的？"

**话术说明：**
- 简洁开放，给用户最大表达空间
- 适用于无法归类到前三类的需求
- AI需要根据用户后续输入判断实际需求，必要时引导用户切换到其他意图
- 保持友好耐心的态度

#### 6.5.5 意图切换话术

**使用场景：** 用户在对话过程中切换意图

**切换话术模板：**
> "好的，我了解您想咨询【新意图名称】相关问题。【对应意图的引导话术】"

**示例：**
- 从健康咨询切换到产品咨询："好的，我了解您想了解健康产品。请问您想为哪方面的健康需求选购产品呢？"
- 从产品咨询切换到健康知识："好的，我来为您介绍健康知识。请问您对什么健康话题感兴趣？"

**话术说明：**
- 先确认切换，让用户感知意图已变更
- 使用新意图的标准引导话术
- 保持对话连贯性，不突兀打断

### 6.6 意图确认交互规范

意图确认功能用于在AI回复未能满足用户需求时，快速引导用户切换咨询意图，提升用户体验和对话效率。

#### 6.6.1 触发条件

| 触发方式 | 说明 | 使用场景 |
|----------|------|----------|
| 用户主动触发 | 点击AI回复下方的"这不是我要的"按钮 | 用户对当前AI回复不满意，希望咨询其他问题 |
| 系统检测触发 | AI检测到用户连续3轮对话与当前意图不符 | 系统自动弹出意图确认提示（可选功能） |

#### 6.6.2 快捷按钮样式规范

| 规范项 | 规格 | 说明 |
|--------|------|------|
| 按钮文字 | "这不是我要的" | 口语化表达，易于理解 |
| 文字样式 | 16px Regular | 灰色（`#999999`），小字链接形式 |
| 图标 | 删除线图标（🚫）或错误图标 | 位于文字左侧，14px，与文字同色 |
| 位置 | AI消息气泡底部 | 位于点赞/踩反馈按钮下方或右侧 |
| 点击区域 | 最小44x44px | 符合无障碍设计规范 |
| 悬停/点击效果 | 透明度变化或下划线 | 提供视觉反馈 |

#### 6.6.3 意图确认弹窗规范

**弹窗结构：**

| 元素 | 规格 | 说明 |
|------|------|------|
| 弹窗标题 | "您是想咨询其他问题吗？" | 22px Bold，居中 |
| 副标题 | "当前回复可能没有解答您的疑问" | 18px Regular，灰色，居中 |
| 主按钮 | "是的，切换意图" | 橙色填充，白色文字，高度56px |
| 次按钮 | "不是，继续聊" | 白色填充，橙色边框，高度56px |
| 关闭方式 | 点击遮罩/向下滑动/点击次按钮 | 多种关闭方式，方便用户操作 |

**弹窗动画规范：**

| 动画属性 | 规格 | 说明 |
|----------|------|------|
| 动画类型 | 从底部滑入 | 符合移动端操作习惯 |
| 持续时间 | 300ms | 流畅但不拖沓 |
| 缓动函数 | ease-out | 先快后慢，自然减速 |
| 背景遮罩 | 黑色半透明（rgba(0,0,0,0.5)） | 突出弹窗，同时保留上下文感知 |
| 遮罩动画 | 淡入（opacity 0→1） | 与弹窗滑入同步 |

#### 6.6.4 用户选择后的反馈

**场景一：用户选择"是的，切换意图"**

| 步骤 | 系统行为 | 用户感知 |
|------|----------|----------|
| 1 | 关闭意图确认弹窗 | 弹窗消失 |
| 2 | 弹出意图选择弹窗 | 显示四个意图选项供选择 |
| 3 | 用户选择新意图 | 弹窗关闭，意图标签更新 |
| 4 | AI发送切换话术 | "好的，我来为您介绍健康知识。请问您对什么健康话题感兴趣？" |

**场景二：用户选择"不是，继续聊"**

| 步骤 | 系统行为 | 用户感知 |
|------|----------|----------|
| 1 | 关闭意图确认弹窗 | 弹窗消失，返回对话 |
| 2 | AI发送安抚话术 | "好的，请继续告诉我您的需求，我会尽力为您解答。" |
| 3 | 保持当前意图 | 意图标签不变，对话继续 |

**场景三：用户点击遮罩关闭弹窗**

| 步骤 | 系统行为 | 用户感知 |
|------|----------|----------|
| 1 | 关闭意图确认弹窗 | 弹窗消失，返回对话 |
| 2 | 不发送额外消息 | 保持对话状态，等待用户继续输入 |

#### 6.6.6 设计原则

| 原则 | 说明 |
|------|------|
| **非侵入性** | 快捷按钮以小字链接形式呈现，不干扰正常对话阅读 |
| **用户主导** | 完全由用户决定是否切换意图，系统不强制引导 |
| **操作简便** | 最多两次点击即可完成意图切换，流程简洁 |
| **上下文保留** | 切换意图不清空历史对话，保持对话连贯性 |
| **反馈明确** | 每个操作都有明确的视觉和文字反馈 |
