P2 用户运营

积分获取（分享/打卡）

用户通过分享推广和每日打卡获取积分，提升用户活跃度和留存

版本
v1.0
2024-01-15

需求概述

提供多种积分获取途径，鼓励用户分享小程序/页面和每日打卡，通过积分激励提升用户活跃度和产品曝光度。 每个获取途径有独立的积分规则和频次限制。

分享推广
分享页面给好友，好友点击后双方获得积分
每日打卡
每日首次进入打卡，连续打卡有额外奖励

功能需求

分享推广模块

FR1 - 分享按钮

P2

在对话页面和积分页面提供分享按钮，支持分享到微信好友/朋友圈。

FR2 - 分享积分奖励

P2

用户分享后，当被分享者点击链接进入，双方获得积分。

积分规则：
  • • 分享者：+20积分/次
  • • 被分享者（新用户）：+50积分（首次）
  • • 每日上限：分享者最多获得100积分/天
  • • 同一被分享者只计算一次

FR3 - 分享追踪

P2

通过URL参数追踪分享来源，记录分享关系链。

每日打卡模块

FR4 - 打卡入口

P2

在积分页面提供打卡按钮，显示今日打卡状态。

FR5 - 打卡积分规则

P2

每日打卡获得积分，连续打卡有额外奖励。

积分规则：
  • • 每日打卡：+10积分
  • • 连续3天：额外+20积分
  • • 连续7天：额外+50积分
  • • 连续30天：额外+200积分
  • • 断签后重新计算

FR6 - 打卡日历

P2

展示当月打卡日历，已打卡日期标记为已完成。

FR7 - 连续打卡提醒

P2

连续打卡即将断签时，在页面显示提醒（如"再打卡2天可获得50积分奖励"）。

数据模型

打卡记录表 (checkin_records)

字段名类型说明
idInteger主键
session_idString用户会话ID
checkin_dateDate打卡日期
consecutive_daysInteger连续打卡天数
points_earnedInteger获得积分
created_atDateTime创建时间

分享记录表 (share_records)

字段名类型说明
idInteger主键
sharer_session_idString分享者ID
visitor_session_idString访问者ID
share_codeString分享码
points_givenBoolean是否已发放积分
created_atDateTime创建时间
