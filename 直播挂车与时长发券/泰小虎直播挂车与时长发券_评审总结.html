泰小虎 · 直播挂车与时长发券 PRD 评审总结

评审对象：泰小虎直播挂车与时长发券 PRD v1.0（评审后已修正为 v1.1）｜评审方式：严格依 PRD 原文，不臆造；图文不一致 / 范围外 / 规范不符一律指出并修正

一、评审概述

本次评审围绕 PRD v1.0 逐条核对端口影响、依赖系统、状态枚举、埋点规范与边界待确认项，共发现 7 类问题，均已修正并重新推送 GitHub。修正动作：重出影响范围图 D-1 与业务流程图 3-1、删改直播间状态枚举、埋点表按《泰小虎埋点表 v2.3》双表结构重写、删除「待确认项汇总」章节及 3 处正文悬空待确认。PRD 线上访问地址：https://ddyuan-spec.github.io/taixiaohu/live-prd.html

二、已明确项（PRD 原文已写清，无需修改）

  • 影响端口：平台端后台（Web）+ 用户端三端（微信小程序 / App(iOS·Android) / H5 内嵌），三端共享小黄车交互、跳转目标不同。
  • 4 个后台模块：直播商品池管理、直播间管理、直播间商品列表、直播数据统计。
  • 用户端小黄车链路：进入直播间 → 小黄车浮层（红角标）→ 商品抽屉 → 点「去看看」→ 跳转确认 → 商品详情下单。
  • 推送 / 同步以火山引擎接口返回为准，失败可点「更新」重推。
  • 依赖既有系统（只读）：商城商品中心《商品表》、火山引擎企业直播 OpenAPI。

三、已裁定范围外项（不在本需求）

项裁定依据
商家端后台 / 火山 SaaS 配置不参与、不碰用户裁定：需求没那么复杂，非火山 SaaS 功能
直播间回放不补用户裁定：泰小虎不用回放功能
老版本 App 无小黄车降级非本期迭代目标本期仅给当前三端新增小黄车

五、风险与建议

  • 接口文档核实：火山直播间状态真实枚举为 直播中(1) / 预告(2) / 回放(3) / 已结束(4)。本需求取前三者；后续新增直播能力须先对接火山企业直播 OpenAPI 确认字段名与枚举，避免再出现"凭通用理解假设枚举"的情况。
  • 埋点落地：v2.3 双表已对齐，建议研发按此表落地，属性字段（room_id / product_id / target / result / operator 等）与商城既有埋点规范保持命名一致。
  • 待确认项治理：生成 PRD 时不应"贪全"塞入范围外或臆造的待确认项。已建立红线——用户裁定不用的功能（商家端、回放、火山 SaaS 配置）一律不添加；边界检查清单只覆盖本期真正涉及的边界。
