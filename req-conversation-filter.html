v1.0

需求概述

在现有对话记录列表基础上，增加更多筛选维度和搜索能力，帮助运营人员快速定位特定对话， 支持按时间范围、意图类型、知识库调用情况、对话轮数等条件筛选。

功能需求

FR1 - 关键词搜索

P2

支持按用户发送的消息内容进行关键词搜索。

搜索范围：
  • • 用户消息内容（user_message字段）
  • • 支持模糊匹配
  • • 不区分大小写
  • • 最少2个字符触发搜索

FR2 - 意图类型筛选

P2

按识别的用户意图类型筛选对话。

产品咨询 症状咨询 用法咨询 价格咨询 副作用咨询 其他

FR3 - 知识库调用筛选

P2

筛选是否调用了知识库的对话。

FR4 - 对话轮数筛选

P2

按会话总对话轮数筛选，支持选择范围（如1-5轮、6-10轮、10轮以上）。

FR5 - 时间范围筛选

P2

支持快速选择（今日、昨日、近7天、近30天）和自定义日期范围。

FR6 - 排序方式

P2

支持按时间、对话轮数、知识库调用次数等多字段排序。

FR7 - 筛选条件组合

P2

支持多个筛选条件同时生效，条件之间为AND关系。
