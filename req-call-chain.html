P1 后台管理

调用链路追踪

单次用户请求的全链路可视化追踪，展示从意图识别到最终回复的完整处理流程

版本
v1.0
2024-01-15

需求概述

为每次用户消息处理生成唯一的Trace ID，追踪该消息在系统中的完整处理链路， 包括意图识别、知识库查询、LLM调用、产品推荐等各个节点的执行情况和耗时， 帮助开发和运营人员快速定位问题环节。

核心价值：
  • • 快速定位问题：当回复异常时，一眼看出哪个环节出错
  • • 性能分析：识别耗时瓶颈，优化系统性能
  • • 数据验证：确认知识库是否正确被调用
  • • 调试辅助：开发阶段追踪处理流程

功能需求

FR1 - Trace ID生成与传递

P1

用户每次发送消息时生成唯一Trace ID，贯穿整个处理链路。

实现要求：
  • • Trace ID格式：txh_ + 时间戳 + 随机字符串
  • • 通过上下文对象在各模块间传递
  • • 关联session_id和conversation_id
  • • 存储到conversation_logs表的trace_id字段

FR2 - 链路节点记录

P1

记录每个处理节点的详细信息。

节点类型 记录内容 耗时记录
intent_recognition 原始意图、识别结果、置信度 ✓
knowledge_query 查询参数、返回结果数、top1相关性 ✓
llm_call 模型名、prompt长度、token消耗 ✓
product_recommend 推荐产品列表、推荐理由 ✓

FR3 - 链路可视化页面

P1

在后台管理提供链路追踪查看页面。

入口位置
对话记录详情页 → 查看链路
展示形式
时间轴 + 节点卡片

FR4 - 节点详情展开

P1

点击任意节点可展开查看详细信息，包括输入参数、输出结果、异常信息等。

FR5 - 耗时分析

P2

显示每个节点的耗时和总耗时，高亮显示耗时较长的节点。
