🗄️ 一、知识库功能设计

知识库是泰小虎智能体的核心数据基础设施，用于存储和管理产品知识、健康知识等内容，支持RAG（检索增强生成）能力。

1.1 知识库架构

组件 说明 技术实现
原始文档 上传的原始文件（PDF、TXT、DOCX等） 文件存储
文档解析 将文档解析为结构化文本 文本提取
知识切片 将长文本切分为语义完整的片段 滑动窗口/语义切分
向量化 将文本转换为向量表示 Embedding API
向量检索 基于向量相似度检索相关内容 余弦相似度/欧氏距离

1.3 知识上传功能

📤 上传流程

  1. 用户进入"知识上传"页面
  2. 选择上传方式：文件上传 / 文本输入
  3. 支持的文件格式：PDF、TXT、DOCX、XLSX
  4. 系统解析文档内容
  5. 用户确认/编辑解析结果
  6. 点击"确认上传"保存到知识库

📝 文本输入模式

用户可以直接在文本框中输入知识内容，系统自动解析并存储。适用于快速添加简单知识点。

输入格式：

产品名称：泰吉眠沙棘茶氨酸

功效：改善睡眠质量

适用人群：失眠人群

1.5 知识检索功能

🔍 检索流程

  1. 接收用户query
  2. 构建增强query（结合用户画像、症状等）
  3. 调用Embedding API生成向量
  4. 在向量数据库中检索相似切片
  5. 返回Top-K结果（默认3条）
  6. 组装上下文传递给LLM
// 检索请求示例
{
  "query": "失眠推荐什么产品",
  "user_context": {
    "age": 42,
    "gender": "女",
    "symptom": "失眠"
  },
  "top_k": 3,
  "filters": {
    "category": "产品功效"
  }
}

// 检索响应示例
{
  "results": [
    {
      "chunk_id": "c001",
      "content": "泰吉眠沙棘茶氨酸主要功效...",
      "score": 0.92,
      "product_id": "PRD001"
    }
  ]
}

📊 三、调用记录功能设计

调用记录功能用于监控和分析泰小虎智能体的运行状态，包括LLM调用记录、知识库调用记录等，为优化和审计提供数据支撑。

3.1 调用记录类型

类型 说明 记录内容
LLM调用 每次LLM API调用 模块名、Token消耗、响应时间、成功率
知识库调用 每次RAG检索 Query、召回数量、关联产品、响应时间
意图识别 每次意图分类 输入、意图类型、置信度

3.2 LLM调用记录结构

// LLM调用记录

3.6 数据存储策略

💾 存储方案

  • 本地文件存储：JSON文件，按会话ID存储
  • 存储路径：data/conversation_logs/{session_id}.json
  • 定期归档：超过30天的会话归档到历史目录
  • 数据导出：支持导出为CSV/JSON格式
⚠️ 生产环境建议：

当前实现使用本地文件存储，适用于中小规模应用。生产环境建议迁移到数据库（PostgreSQL/MongoDB）或时序数据库（InfluxDB），支持更高效的查询和分析。
